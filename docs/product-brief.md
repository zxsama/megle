# Megle Product Brief

## 目标

Megle 是一个 Windows 优先的本地媒体浏览与管理工具。它索引用户已有的目录结构，提供类似 Eagle 的高效率浏览、预览、筛选和整理体验，但不把原图导入到私有图库目录。

核心原则：

- 只索引已有 Windows 目录，不复制原始图片或视频。
- 左侧目录树映射真实文件夹，不做虚拟文件夹作为首版主模型。
- 标签、评分、备注等用户元数据只保存在 Megle 数据库中。
- 允许对真实文件执行重命名、移动、删除，但必须有明确确认、操作日志和可恢复策略。
- UI 交互体验参考 Eagle 的高密度媒体浏览方式，视觉可以采用克制的液态玻璃风格。
- 首版只做 Windows 桌面版，但架构必须保留未来 Web/Docker 部署路径。
- 浏览器扩展以后以插件形式支持，首版先设计插件扩展点，不优先实现浏览器扩展。

## 规模假设

Megle 必须按几百万级媒体文件设计，而不是按几万张图片设计。

目标工作负载：

- 1 到 5 个本地或外接磁盘根目录。
- 单个库 100 万到 500 万媒体文件。
- 支持 HDD、SSD、移动硬盘、NAS/网络盘，但性能目标优先针对 SSD。
- 文件格式覆盖常见图片和视频，扩展格式通过解码器插件逐步增强。

## 首版范围

首版必须完成：

- 添加已有目录为索引根。
- 递归扫描目录，识别图片和视频文件。
- 生成并缓存缩略图。
- 左侧真实目录树。
- 中间虚拟滚动媒体网格。
- 右侧属性面板。
- Space 或 Enter 快速预览。
- 键盘左右切换图片/视频。
- 标签、评分、收藏。
- 文件名、路径、标签、评分、格式、尺寸、日期筛选。
- 对真实文件执行重命名、移动、删除到回收站。
- 文件系统变更监听和后台增量更新。

暂不优先：

- 云同步。
- 多人协作。
- 浏览器扩展保存图片。
- AI 搜索。
- 跨设备同步。
- 完整 DAM 权限系统。

## 非目标

- 不反编译或复用 Eagle 的源码、资源、协议或私有实现。
- 不创建 Eagle 兼容库格式。
- 不强制整理用户文件目录。
- 不在首版把所有高级格式做到完美。格式支持采用分层解码器和插件策略。

## 关键体验指标

- 应用启动后，已有索引库应在数秒内可浏览。
- 网格滚动不因图片数量增长而变慢。
- 缩略图缓存命中时，应快速显示当前视口内容。
- 切换上一张/下一张时，UI 响应不能被原图解码阻塞。
- 当前图附近的图片应被后台预取。
- 大图、超大图、视频首帧必须先显示可用预览，再异步加载高质量内容。
