# Performance Plan

## 性能目标

Megle 的性能目标按百万级媒体库定义。

建议首版验收指标：

- 已索引库启动到可浏览：小于 3 秒。
- 打开普通目录的首屏查询：小于 100ms。
- 5M 元数据热路径查询：小于 100ms。
- 5M 库后台写入时前台查询 p99：小于 50ms。
- 5M 库 200 项分页 API 查询+序列化 p99：小于 20ms。
- 500k 已存在文件首轮遍历+写入：小于 5s。
- 缓存命中缩略图首屏显示：小于 300ms。
- 图片 `grid` 缩略图短边 320px，后台生成吞吐目标：大于 100 张/秒。
- 小于 320px 短边的图片应跳过独立 `grid` 缩略图生成。
- 生成类缩略图统一使用 WebP，避免 JPEG/PNG/AVIF 混用导致缓存和前端逻辑复杂化。
- 文件重命名/移动/软删除一致性测试必须保持 0 mismatch。
- 缓存 WebP 预览切换 p99：小于 50ms，状态错位必须为 0。
- 网格滚动保持 60fps 优先，最低不低于 30fps。
- 切换上一张/下一张时，UI 状态更新小于 50ms。
- 缓存预览图命中时，大图预览小于 150ms 出图。
- 后台索引、缩略图生成不能阻塞 UI。

这些指标需要在 10 万、100 万、500 万文件三个数据集上分别压测。

## 关键性能策略

### 1. 永不全量加载

前端不能一次性读取一个目录或全库的所有媒体项。

所有列表 API 必须分页：

- `limit`
- `cursor`
- `sort_key`
- `direction`
- `filters`

使用 keyset pagination，不使用大 offset。

### 2. 虚拟滚动

网格只渲染可见区域和少量 overscan。

要求：

- item 尺寸稳定。
- 缩略图加载不能改变布局。
- 滚动中降低非必要阴影、模糊和动画。
- 液态玻璃效果只用于工具栏、侧栏、弹层，不覆盖大面积缩略图网格。
- 不要用一个未分段 DOM 滚动容器承载 5M 全库视图。
- 超大结果集必须做 segmented/windowed browsing。

### 3. 多级缓存

缓存层：

- SQLite page cache。
- Core 内存 LRU：查询结果、目录 children、媒体元数据。
- Thumbnail disk cache。
- UI image cache。
- Preview decode cache。

内存缓存必须有上限，例如：

- metadata/query cache: 128MB 到 512MB。
- preview image cache: 512MB 到 2GB，按机器内存自适应。
- background worker queue: 有界队列，避免扫描时吃满内存。

### 4. 优先队列

所有重任务都进入优先队列。

优先级从高到低：

1. 当前预览项。
2. 当前视口缺失缩略图。
3. 当前选中项前后预取。
4. 用户刚打开的目录。
5. 后台全库补齐。

用户交互任务必须能取消或抢占后台任务。

### 5. 解码器隔离

图片和视频格式很多，解码失败很常见。

要求：

- 解码任务在 worker 中运行。
- 单个坏文件不能拖垮 Core。
- 对可能崩溃的外部解码器使用子进程隔离。
- 对超大文件设置时间、内存、尺寸限制。
- 解码失败要入库记录状态，避免反复失败重试。

### 6. 查询索引

SQLite 可以支撑百万级元数据，但需要纪律：

- 开启 WAL。
- 批量写入使用事务。
- 高频查询都有复合索引。
- 后台写入节流，避免拖慢前台读取。
- 定期 analyze。
- 大文本搜索可使用 FTS5。
- 热路径查询必须由最终排序索引驱动。
- 避免先 join/filter 出大量候选再排序。

当全库搜索、相似图、AI 搜索增长后，再引入独立索引：

- Tantivy 用于全文搜索。
- 向量库用于 AI embedding。
- pHash BK-tree 或专用表用于相似图。

## 大规模数据集测试

需要准备三组测试库：

- 10 万文件：开发日常基准。
- 100 万文件：性能门槛。
- 500 万文件：极限验证。

测试内容：

- 初始扫描耗时。
- 重启后打开库耗时。
- 目录树展开耗时。
- 网格滚动帧率。
- 缩略图缓存命中/未命中表现。
- 切图延迟。
- 搜索延迟。
- 文件重命名/移动/删除一致性。

必须提供自动化脚本生成假目录和小图片/视频样本，不能靠人工数据测试。

## 风险

- 几百万小缩略图文件会放大 NTFS 元数据压力。
- NAS watcher 事件可能不可靠。
- RAW/PSD/HEIC/视频格式支持会带来依赖、授权和稳定性问题。
- 液态玻璃大面积 blur 会影响滚动性能。
- 真实文件移动/删除失败场景很多，必须有操作日志。
