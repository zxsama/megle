# SQLite 1M Benchmark

Date: 2026-05-16

## Purpose

Validate the first Phase 0 performance gate: SQLite schema shape, WAL mode, bulk insert, indexes, keyset pagination, and representative media-list queries at 1 million media rows.

This benchmark uses Python stdlib `sqlite3` because the Rust workspace has not been created yet. It is a first gate, not the final Rust Core benchmark.

## Environment

- OS: Windows 11 `10.0.26200`
- Python: `3.13.3`
- SQLite: `3.49.1`
- Visible CPU threads: `32`
- Hardware details: CIM queries were denied by sandbox permissions.

## Important Finding

The first version of `recent_images` was slow:

```sql
SELECT f.id, f.name, f.mtime, m.width, m.height
FROM files f
JOIN media m ON m.file_id = f.id
WHERE m.kind = 'image'
ORDER BY f.mtime DESC, f.id DESC
LIMIT 200;
```

SQLite planned it by scanning `media(kind)`, joining to `files`, then building a temporary sort tree. On the 1M dataset this measured around `236-311 ms`.

The optimized version drives from `files(mtime DESC, id DESC)` and verifies media kind per candidate:

```sql
SELECT f.id, f.name, f.mtime,
       (SELECT m.width FROM media m WHERE m.file_id = f.id),
       (SELECT m.height FROM media m WHERE m.file_id = f.id)
FROM files f
WHERE EXISTS (
    SELECT 1
    FROM media m
    WHERE m.file_id = f.id
      AND m.kind = 'image'
)
ORDER BY f.mtime DESC, f.id DESC
LIMIT 200;
```

This measured around `0.208 ms` p50.

Decision:

- Hot browse queries must be driven by the final sort index.
- Avoid join plans that filter large candidate sets and then sort.
- For later filters where candidate density is low, consider denormalized hot fields or materialized browse indexes.

## Result

The first SQLite gate passes at 1M synthetic media rows for metadata browsing and search.

Caveats:

- This is synthetic metadata, not real filesystem scanning.
- This is warm-cache query performance.
- This does not test thumbnail generation or media decoding.
- This does not test concurrent writer/reader pressure.
- This uses Python `sqlite3`, not the final Rust `rusqlite` implementation.
